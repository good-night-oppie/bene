BENE 2.0 Research Synthesis

This document is the paper grounding for the BENE 2.0 redesign. The corpus is the eddie-agi-kb gold lists (~100 entries across the nine mined lists; two further lists — agent-fun-research and apple-em — were excluded up front, see rubric); after relevance selection and deduplication, 48 citations are mined here and mapped onto the five BENE 2.0 pillars: (1) KAOS-parity core, (2) Evolution engine, (3) Memory & context OS, (4) Harness-engineering layer, (5) Trust & Experience. Each citation records the exact KB entry directory, read depth, the core idea, and the BENE subsystem it informs.

Selection rubric

  • Per-list relevance picks targeting the five pillars. Within each gold list, entries were selected for direct relevance to one of the five BENE 2.0 pillars; each cited paper is assigned exactly one primary pillar.
  • deep-read = full transcript read (transcripts under ~250 lines were read in full; longer ones read to ~250 lines, slides consulted only when transcripts were ambiguous).
  • skimmed = title/slides-level only (kept when the title-level idea was still pillar-relevant, e.g. as a dedupe representative).
  • Dupes deduped. Several lists contain byte-identical or near-identical re-ingests of the same videos (verified via md5/cmp/diff); only the canonical (fullest-transcript) entry per unique source is cited.
  • Deliberately skipped:
  • The agent-fun-research list (out of scope for the redesign).
  • The apple-em list (empty — no entries to mine).
  • Duplicate entries across all lists (e.g. agent-context-memory 06–09 byte-identical to 01–04; agent-loop-papers 06–10 title-duplicates of 01–05; agent-multi-prompt-opt 04–06 duplicates of 01–03; agent-skills-mgmt 05–08 byte-identical to 01–04; agent-kv-cache 02 symlink-duplicate of 01; vibe-coding pairwise duplicates).
  • Training-time / out-of-scope papers in agent-auto-opt-papers: skill0 (near-duplicate of Skill1; parameter internalization), CoEvolve (RL co-evolution), SkillForge (domain-specific; mechanism covered by SkillClaw + Trace2Skill), FedTextGrad (federated multi-tenant; BENE is local-first single-tenant).
  • Series-internal near-duplicates and commentary in the harness-engineering blog corpus (OpenAI pt 5, Schmid pt 2, LangChain pts 2/3/5, Claude Code pts 2/3, Latent Space commentary, Cursor series, index files) and news-event coverage in vibe-coding (Claude Code source-leak pair).

Citations by list

agent-auto-opt-papers

Meta-Harness: End-to-End Optimization of Model Harnesses (Stanford, arXiv 2026-03) (entry: 01_[Harness自动优化]01期-Meta-Harness-斯坦福爆火论文-别只卷模型，来“自动发明模型外壳”（简介附原文链接）, deep-read) — Treats the harness (what to store, how to retrieve, context assembly, turn orchestration, termination) as a searchable design space and optimizes it end-to-end; weights set the theoretical ceiling, the harness decides how much of it you actually reach. Informs: bene/metaharness/search.py — formalize the harness genome (memory/retrieval/context-assembly/termination policies) as the explicit search space for mh_search instead of prompt-only candidates. Pillar: 2.

SkillX: Automatically Constructing Skill Knowledge Bases for Agents (entry: 02_[基于skill的Agent进化]01期-SkillX-将强Agent的成功轨迹构建为分层Skill库，实现可迁移的泛用性进化, deep-read) — Distills a strong agent's successful rollouts into a 3-tier skill KB (Planning / Functional / Atomic skills) with iterative merge+filter refinement (anti-duplication, anti-tool-hallucination) and unlock-time retrieval, transferable to weaker base models. Informs: BENE trace->skill distillation: make SkillStore hierarchical (route-map / reusable sub-procedure / per-tool gotchas) with Add/Modify/Keep update semantics and a two-layer quality filter before writeback. Pillar: 2.

From Procedural Skills to Strategy Genes: Experience-Driven Test-Time Evolution (GEP — Gene Evolution Protocol; EvoMap x Tsinghua) (entry: 03_[基于经验的Agent优化]01期-EvoMap和清华的合作论文-把经验编码成控制信号，让进化真正生效（简介附原文链接）, deep-read) — Encodes experience as ~230-token "strategy genes" (match-signal + summary + strategy steps + AVOID warnings) organized by control logic, not documentation; GEP loop Scan->Signal->Intent->Mutate->Validate->Solidify makes genes versioned, evolution-ready objects. Key finding: explanatory doc content (overviews, examples) actively dilutes and hurts; control-logic density is what works. Informs: BENE skill/memory representation: store dense control-signal genes (with gene/capsule/event = unit/validated-execution/immutable-evolution-log) rather than long SKILL.md-style docs injected wholesale. Pillar: 2.

Trace2Skill: Distill Trajectory-Local Lessons into Transferable Agent Skills (Alibaba Qwen) (entry: 04_[基于skill的Agent进化] 阿里Qwen新论文-Trace2skill-分层整合从不同轨迹中得到的修改建议, deep-read) — Parallel per-trajectory analysts emit skill "patches" (success analyst single-pass; error analyst builds an evidence chain with verified root cause + minimal-fix validation), then prevalence-weighted hierarchical conflict-free merge into one static skill doc — avoiding sequential-update drift; beats retrieval-style experience banks. Informs: BENE trace->skill pipeline mechanics: patch-based parallel consolidation over engrams, evidence-chain failure attribution, and "distilled static skill beats RAG-over-traces" as a design bet. Pillar: 2.

SkillClaw: Let Skills Evolve Collectively with Agentic Evolver (Alibaba DreamX) (entry: 05_[基于skill的Agent进化]阿里DreamX团队论文-SkillClaw-聚合多用户轨迹得到稳定进化方向，夜间验证保证有效性, deep-read) — Aggregates multi-user trajectories grouped by invoked skill (a "natural ablation" exposing skill behavior boundaries); an Evolver picks Refine/Create/Skip with success sessions as invariants; candidate skills deploy only after nightly real-environment A/B validation beats the incumbent (monotonic deployment). Informs: BENE consolidation/dream cycle: nightly validation gate before skill writeback to the shared library — skill plasticity with a no-regression guarantee across agents. Pillar: 1.

AHE: Agentic Harness Engineering (Fudan/PKU) (entry: 08_[Harness自动优化]02期 _ 复旦爆火论文-AHE-可观测体系驱动Harness自进化, deep-read) — Makes the full harness (prompt, tools, middleware, skills, memory) a learnable surface via three observability layers: component (file-based editable components), experience (hierarchical trace digests w/ progressive disclosure), decision (Change Manifests predicting fixes/regressions, verified next eval round, file-level rollback). Factual/executable components transfer; system-prompt-only edits score below seed. Informs: BENE harness-evolution loop discipline: evidence->prediction->verify->rollback per edit, component-decomposed candidates for the Surrogate Verifier, and de-prioritizing prompt-text mutations vs memory/tool mutations. Pillar: 2.

Ctx2Skill — From Context to Skills: Can Language Models Learn from Context Skillfully? (Tsinghua et al.) (entry: 10_[skill进化] 清华爆火论文-Ctx2Skill-零标注自博弈，从文档提取技能, deep-read) — Zero-label self-play over a single document: Challenger generates rubric-scored probe tasks, Reasoner answers with current skills, Judge grades, Proposer diagnoses gaps and Generator rewrites the skill doc; cross-time replay on hard+easy probes selects the best historical skill version to avoid adversarial collapse (the last version is often not the best). Informs: BENE skill bootstrap when no traces exist yet (manuals, rule docs) + falsifiable eval probes and replay-based skill-version selection for the consolidation step. Pillar: 1.

Autogenesis: A Self-Evolving Agent Protocol (AGP) (NTU/Stanford/Princeton) (entry: 14_[Harness自动进化] 04期 _ Autogenesis：南洋理工提出AGP协议，让Agent Harness进化可审计，LeetCode C++ 近满分, deep-read) — Two-layer protocol for governing self-modification: RSPL registers five resource types (prompt/agent/tool-skill/environment/memory) as passive, versioned objects with lineage in a registry; SEPL runs Reflect->Select->Improve->Evaluate->Commit so every change is a candidate that must pass evaluation before commit, with rollback — turning self-evolution from black-box edits into auditable system events. Informs: BENE Nexus (SQLite) layer: register evolvable harness resources with version lineage, candidate/commit/rollback semantics — provenance and auditability of every self-modification. Pillar: 5.

AEVO: Harnessing Agentic Evolution (HKUST-GZ, DeepWisdom, et al.) (entry: 15_[Harness自动优化] 05期 _ 港科大爆火论文-AEVO：让MetaAgent接管进化机制，破解智能体长程进化失控问题, deep-read) — Treats the long-horizon evolution process itself as an interactive environment: accumulated candidates/feedback/failures/cost are state held in a harness with evaluator isolation, and a MetaAgent edits the future search mechanism (selection policy, feedback format, skills, budget) rather than emitting candidates. ~26% relative gain; removing the harness boundary caused reward hacking in 2/3 runs. Informs: BENE meta-harness search: add a process-level MetaAgent that diagnoses stagnation/repeated failure and rewrites search mechanics mid-run, plus hard isolation of the verifier from candidate-generating agents. Pillar: 2.

Skill1 (Meituan) — one reward signal split three ways to train the full skill-usage pipeline (entry: 07_[skill进化]-美团新论文-skill1-一个信号拆三份，同时学习skill使用全流程, skimmed) — Title-level: decomposes a single training signal into three to jointly learn skill retrieval/selection/application; companion to Skill0 (internalizing skills into model parameters via ICRL). Informs: Marginal for BENE (training-time skill internalization; BENE is a no-finetune orchestration layer) — noted as the dedupe representative of the 06/07 Meituan pair. Pillar: 2.

Continual Harness (Google x Princeton) — online harness rewriting while playing Pokemon (entry: 09_[Harness自动优化] 03期 _ 谷歌×普林斯顿新论文-一边玩宝可梦，一边自动改写Agent Harness, skimmed) — Title-level: rewrites the agent harness on-line during a long-horizon task rather than between benchmark rounds — continual, in-episode harness adaptation. Informs: BENE long-running agents: candidate idea for in-episode harness mutation (vs BENE's current between-generation mh_search), referenced by AHE/AEVO transcripts as "Continual Harness". Pillar: 2.

MUSE-Autoskill (ByteDance) — skill lifecycle management: auto create/test/maintain (entry: 12_[Skill进化] 06期 _ 字节MUSE-Autoskill：给Agent装上Skill生命周期管理，自动创建、测试、维护Skill, skimmed) — Title-level: gives the agent a full skill lifecycle manager — automatic creation, testing, and maintenance of skills as first-class managed artifacts. Informs: BENE SkillStore lifecycle: skills need birth/test/deprecate states, complementing SkillClaw's validation gate. Pillar: 1.

agent-context-memory

AgentSwing: Adaptive Parallel Context Management Routing for Long-Horizon Web Agents (Alibaba Tongyi Lab) (entry: 01_[Agent上下文管理] 阿里通义新论文-AgentSwing-动态选择最合适的上下文管理策略（简介附链接+总结）, deep-read) — At a context-length threshold, fork parallel branches each with a different context strategy (keep-last-N / summarize / discard-all), roll each K lookahead turns against the env, then route to the branch with the best real continuation (k=3 sweet spot; 39.5->60.0 BrowseComp). Informs: Checkpoint/restore as a context-recovery primitive: branch-fork + lookahead + route in bene/ccr/runner.py loop; context strategy becomes a runtime routing decision (and a strategy gene for mh_search), not a fixed compression rule. Pillar: 3.

MemGAS — From Single to Multi-granularity: Long-term Memory Association and Selection for Conversational Agents (ICLR 2026, USTC/CityU/Huawei) (entry: 02_[Agent记忆] ICLR26-MemGAS-多粒度记忆关联与自适应选择, deep-read) — Store memories at 4 granularities (session/turn/summary/keyword), link them via GMM-filtered association graph; an entropy router weights granularities per query, then personalized-PageRank propagation from seed nodes + LLM filter retrieves associatively (~10% RAM overhead). Informs: Engrams/trace-RAG layer: index execution traces at multiple granularities with an association graph and entropy-based granularity routing instead of flat single-chunk vector top-K in memory_search. Pillar: 3.

RF-Mem — Evoking User Memory: Personalizing LLMs via Recollection-Familiarity Adaptive Retrieval (ICLR 2026 poster, DUT/CityU/Huawei/USTC) (entry: 03_[Agent记忆] ICLR26-RF-Mem-熟悉度驱动的快慢双路径记忆检索, deep-read) — Probe retrieval computes mean similarity + entropy of candidates; confident queries take the fast one-shot top-K (familiarity) path, uncertain ones take a bounded slow recollection loop (k-means cluster centroids alpha-mixed into reformulated queries, beam/fanout/max-rounds caps). Informs: Adaptive retrieval controller for BENE memory/trace search: uncertainty-gated compute spend, near-one-shot latency on easy queries, deeper evidence reconstruction only when entropy is high; pluggable atop any memory index (pairs with tier router philosophy). Pillar: 3.

MemCoE — Learning How and What to Memorize (ACL 2026, USTC/CityU/DUT/XJTU) (entry: 04_[Agent记忆] ACL26-MemCoE-先学怎么记，再学记什么, deep-read) — Two-stage memory evolution: (1) induce a global natural-language memory-update guideline via batched textual-gradient critiques of good vs bad trajectories; (2) GRPO RL with guideline-alignment process rewards + answer reward learns what to write/merge/deprecate. Guideline transfers across LLM backbones; 74% vs 51% preference retention after 10 update rounds. Informs: Memory write/consolidation (dream) policy: make the update guideline an explicit evolvable artifact — BENE's mh_search textual-evolution loop can breed the guideline, and per-update process checks guard against overwrite-drift in long-lived agent memory. Pillar: 3.

GAM — General Agentic Memory via Deep Research (BAAI/Renmin U/PKU/PolyU HK) (entry: 10_[Agent记忆] GAM-即时编译式记忆，轻量索引加运行时研究, deep-read) — JIT-compiled memory: offline Memorizer keeps lightweight memos + full headered pages (no lossy ahead-of-time compression); online Researcher runs plan -> parallel search (vector + BM25 + page-id) -> integrate -> reflect loops to compile an optimized context per request. RAG scores 0 on Ruler MultiHop Tracing, GAM >90; small model suffices for Memorizer, Researcher needs a strong one. Informs: Engrams architecture: keep full raw traces in the content-addressable blob store with a lightweight memo index, and serve "Other Memory" via a deep-research retrieval agent over traces instead of precomputed summaries; tier-route Memorizer to cheap models, Researcher to strong ones. Pillar: 3.

agent-loop-papers

ReAct (Reasoning + Acting) (entry: 01_Agent Loop智能体循环（1）ReAct：思考后行动，Agent领域的基石, deep-read) — Interleave explicit natural-language thought, tool action, and observation in a loop; verbalized reasoning cuts hallucination and makes failures inspectable. Clear tool descriptions are the load-bearing interface. Informs: BENE ccr/runner.py agent loop + ccr/tools.py registry: keep thought/action/observation as first-class trace events (Engrams), and treat tool-description quality as a designed surface (pillar-4 tool design, loop transparency). Pillar: 4.

Plan-and-Execute / PEARL (Plan-and-Execute over Long Documents, ACL 2023) (entry: 02_Agent Loop智能体循环（2）Plan&Execute：先规划再执行，也是Agent领域基石, deep-read) — Instead of stuffing a long document into the window, the model first generates an executable plan decomposing the task into small steps over document parts, then executes steps sequentially, feeding intermediate results back to revise later steps. Informs: BENE context-strategy selection for long inputs: a planner-executor mode that decomposes long-context work into planned sub-reads with result feedback, rather than single-shot ingestion; composable with trace-RAG retrieval. Pillar: 3.

AdaPlanner (NeurIPS 2023) (entry: 03_Agent Loop智能体循环（3）AdaPlanner：规划反思+总结技能，也是Skills的前身, deep-read) — Closed-loop adaptive planning: code-style plans with per-subgoal assertions; assertion pass/fail triggers in-plan refinement or full out-of-plan replanning, and successful plans are distilled into a skill memory reused as few-shot exemplars. Informs: BENE 2.0 evolution engine: trace-to-skill distillation (successful execution traces -> SkillStore exemplars) plus assertion-gated replanning as a verifier hook in the runner; explicitly the precursor of Skills. Pillar: 2.

ReCAP (Recursive Context-Aware Planning, NeurIPS 2025) (entry: 04_Agent Loop智能体循环（4）NeurIPS25-ReCAP：规划+逐步细化，专门用于Agent长周期任务, deep-read) — For long-horizon tasks: decompose the full subtask list up front, execute one step, then progressively refine the rest; structurally re-inject the parent plan into the current context on recursive return, with active-prompt size capped so cost grows linearly with depth. Informs: BENE memory & context OS: anti-context-drift via plan re-injection across hierarchy levels and bounded active context for long-horizon agents — directly relevant to checkpoint/restore-aware context reconstruction. Pillar: 3.

RLM (Recursive Language Models, 2026 preprint) (entry: 05_[Agent Loop] 05期-MIT爆火论文-RLM-程序化递归调用，完成超长上下文任务（简介附原文链接）, deep-read) — Store the arbitrarily long prompt as a variable in a persistent REPL environment; the model emits code to inspect/slice/transform it and recursively calls sub-RLMs, with only constant-size metadata (length, short prefixes) appended to history — never the raw text — to prevent window pollution. Informs: BENE long-context loops + KV/window hygiene: an environment-as-memory pattern (SQLite Nexus as the persistent state, sub-agent recursion with metadata-only history) for near-unbounded input/output without context pollution. Pillar: 3.

agent-multi-prompt-opt

GEPA (Genetic-Pareto Evolutionary Prompt Adaptation) (entry: 01_工作流Agent多Prompt联合优化（1）GEPA：超越GRPO，让Prompt 像基因一样进化, deep-read) — Evolves prompts as genes (selection/crossover/mutation) filtered by a Pareto-optimal multi-objective frontier; natural-language self-reflection on failed runs extracts high-level rules that steer targeted mutations instead of blind trial-and-error. Informs: BENE meta-harness search loop (bene/metaharness/search.py): reflection-driven mutation of harness/prompt candidates plus a multi-objective Pareto frontier (quality vs cost/speed) for candidate survival; reflection notes should be persisted as Engrams/trace-derived rules. Pillar: 2.

MIPRO (Multi-prompt Instruction Proposal Optimizer, Stanford/Berkeley, DSPy) (entry: 02_工作流Agent多Prompt联合优化（2）MIPRO：贝叶斯优化探索最优Prompt提示词组合, deep-read) — Jointly optimizes all module prompts in a multi-step LLM program: bootstraps few-shot demos from score-thresholded successful execution traces, proposes grounded instructions using dataset/control-flow context, and uses Bayesian surrogate optimization with mini-batch eval to solve cross-module credit assignment. Informs: BENE trace->skill distillation and meta-harness candidate proposal: mine successful traces in bene.db as bootstrapped demos/skills, and use a surrogate model (cheap mini-batch scoring) to pick prompt/strategy combinations instead of exhaustive full-benchmark runs. Pillar: 2.

ADOPT (textual-gradient multi-prompt optimization with Shapley budget allocation) (entry: 03_工作流Agent多Prompt联合优化（3）ADOPT：超越GEPA，像训练模型一样训练Prompt, deep-read) — Trains prompts like model weights: learns step->output dependencies from successful traces, converts error cases into a global textual loss/gradient, decomposes it into per-step local gradients (pluggable single-prompt optimizers), and allocates optimization budget by Shapley-value step contribution; handles loops/non-DAG pipelines. Informs: BENE Surrogate Verifier (bene/metaharness/verifier.py) structured failure diagnostics: turn benchmark failures into per-component textual gradients with Shapley-style credit assignment so the evolution engine spends mutation budget on the harness component actually dragging scores down. Pillar: 2.

agent-skills-mgmt-papers

GraSP (Graph-Structured Skill Compositions for LLM Agents, Tencent) (entry: 01_[Agent skill编排]01-腾讯GraSP-编排skill graph，规划Agent行动路径, deep-read) — Don't feed retrieved skills as flat prompt text: compile them into an executable DAG (state/data/order edges) validated against skill schemas (preconditions/effects/IO), then repair failures locally with 5 budgeted operators (rebind, insert-prior-skill, substitute, rewire, bypass) instead of global replanning, preserving already-verified nodes. Informs: BENE harness layer: skill metadata schema in ccr/tools.py as compilable orchestration material (preconditions/effects/IO bindings); DAG compile step before the ccr/runner.py loop; typed-failure events + budgeted local repair that protect checkpointed/verified progress (loop guards, confidence-based fallback to plain ReAct). Pillar: 4.

SkVM (Skill Virtual Machine — Revisiting Language VMs for Skills Across Heterogeneous LLMs and Harnesses, SJTU) (entry: 02_[Agent skill编译] 上海交大爆火论文-SkVM-把Agent Skill从提示词变成可编译的系统组件, deep-read) — Treat a skill as a natural-language program, not context: AOT-compile at install time (profile model+harness against 26 primitive capabilities, compensate or substitute paths, bind environment deps, extract DLP/ILP/TLP parallelism), then JIT-recompile from structured failure logs with rollback, and solidify stable code templates into gated direct-call functions (+15.3% avg completion, ~40% token savings). Informs: BENE skill runtime: per-tier skill compilation in router/tier.py (same skill adapted per provider — local vs claude_code vs agent_sdk), environment-binding scripts before agent runs, failure-log-driven recompile with rollback (pairs with metaharness/verifier.py diagnostics), and code solidification for repeated tool calls. Pillar: 4.

SKILLGRAPH (USTC / Alibaba / NUS) (entry: 03_[Agent skill编排]02-阿里最新论文-SKILLGRAPH-让技能关系图和模型参数共同进化, deep-read) — Distill skills from success AND failure trajectories into a graph (prerequisite / enhancement / co-occurrence edges) that co-evolves with GRPO policy training: nodes get inserted/merged/split/deprecated by usage stats, edges strengthen on successful traversal and decay otherwise; retrieval = seed + bidirectional BFS + topological sort, with curriculum-style progressive layer unlocking (7B beats GPT-4o on ALFWorld by 42.6 pts). Informs: BENE evolution engine: trace->skill distillation in metaharness/search.py should produce a skill RELATION graph from Engrams (traces), not a flat library — edge reinforcement/decay as the graph analog of strategy-gene selection, and topologically-ordered skill retrieval for trace-based RAG. Pillar: 2.

MMSkills (Towards Multimodal Skills for General Visual Agents, SJTU / Xiaohongshu / SEU) (entry: 04_[Agent Skills] 视觉Skill来了-上海交大新论文-MMSkills让Agent学会判断屏幕状态, deep-read) — Skills must encode WHEN, not just HOW: each skill packs a textual procedure + runtime State Cards (when-to-use/not, visible cues, verification cues) + keyframe bundles, loaded via a "branch" sub-check (gated view selection + branch planning) that returns structured guidance without polluting the main agent's context or anchoring it to stale screenshots; cuts blind repeated actions 21.8%->6.2%. Informs: BENE harness "agent senses" + verification bottleneck: skill format should carry applicability conditions and verification evidence; isolate evidence-checking into a side branch (maps to a Surrogate-Verifier-style subagent in metaharness/verifier.py) so the main loop acts only on live state — also a guard against context pollution from heavy skill payloads. Pillar: 4.

agent-theory-research

LLMs Get Lost in Multi-Turn Conversation (Microsoft Research + Salesforce, arXiv 2025-05; "Lost in Conversation" / sharded-instruction evaluation) (entry: 03_ICLR26 Best Paper-多轮对话一旦走错，Agent为什么很难回头？上下文污染让模型迷失, deep-read) — Sharding single-turn tasks into per-turn info shards drops performance ~39% vs full prompt; decomposed, aptitude falls only ~16% while unreliability rises ~112% — premature answers pollute context (answer bloat 20-300%, lost-in-middle-turns), and recap strategies or temperature=0 cannot fully recover; consolidating all requirements into a fresh conversation does. Informs: BENE context OS: context-pollution detection + consolidate-then-restart recovery — auto-summarize accumulated requirements from the trace and respawn a clean-context agent (checkpoint/restore + trace Engrams) instead of patching a derailed thread; also discourage early full-answer commitment in the runner loop, and adopt aptitude-vs-unreliability as an eval metric for multi-turn harness candidates. Pillar: 3.

Seeing but Not Believing: Probing the Disconnect between Visual Attention and Answer Correctness in VLMs (UIUC + Amazon + Penn State; VEA — Visual Evidence Augmentation) (entry: 02_[Agent理论] 02 _ ICLR26-亚马逊&UIUC爆火论文-视觉语言模型分析-为什么有时VLM看到了证据，但还是答错？如何非训练矫正, deep-read) — VLMs often attend to the correct image evidence in deep layers yet still answer wrong (text-dominance suppresses visual signal); training-free fix: profile which layers localize evidence (AUROC vs evidence boxes), build a patch-level evidence map (denoise + Gaussian smooth), re-weight the image to highlight evidence and dim background, then re-ask — +5.7 ExactMatch avg across 8 models, robust under noise/occlusion. Informs: BENE context OS: "evidence present ≠ evidence used" — adaptive retrieval/recovery should re-highlight already-in-context evidence (restructure, foreground, dim irrelevant context) as a cheap inference-time intervention before re-retrieving or retrying; the model's own internal signals can be elicited and verified by whether the intervention changes the answer. Pillar: 3.
