# BENE 2.0 — 架构全景 (Architecture)

> **BENE** — **B**reeding-program (繁育计划) · **E**volutionary (进化) · **N**exus (核心枢纽) · **E**ngrams (记忆印记) — 一套为 AI 编程写手量身定制的、拥有 Bene Gesserit 基因的控制引擎 (harness)。
>
> 2.0 版本的终极信条：*世间万物，皆为印记 (everything is an engram)。* 在五大能力支柱之下，铺着一层带强类型、只许追加、用血脉谱系 (provenance) 串联起来的印记底座；凡是敢进化的东西，全得过击杀闸门 (falsifiable gates)；凡是敢动手做事的 Agent，全得爬自主权阶梯 (autonomy ladder)。

设计背后的血泪账 (每一个拍板都得拿科学文献 / 压缩理论 / 工程现实来挨个过堂)：`DESIGN-RATIONALE.md` (D1–D10)。
给论文党看的证据：`../research/SYNTHESIS.md` (48 篇被我们榨干的论文)。给同行看的降维打击报告：`../research/GAP-AUDIT.md`。
给码农看的施工图纸 (DDL 结构 + API 签名 + 搬家计划)：`KERNEL-SPEC.md`。

这个内核就是那个 *Nexus (枢纽)* 在物理世界里的真身：一个把存储协议包在底下的 SQLite 文件，它既装下了所有特种兵现在干活的摊子，**又**装下了他们所有经历的、不同压缩级别的记忆，而且全靠血脉连线死死串在一起。跟《沙丘》原著里的黑话现在有了极其硬核的结构对应关系：

| 沙丘原著 | 1.0 版本 (纯靠嘴吹) | 2.0 版本 (真枪实弹的机制) |
|---|---|---|
| 其他记忆 (Other Memory) | 能被搜索的运行日志 | 印记底座：一部把运行历史从原片压缩到本能的完整阶梯，而且能顺着血脉往上往下查 |
| 传教使团 (Missionaria Protectiva) | 技能和记忆的分发 | 在出事之前，把 tier-3 级别的肌肉记忆 (技能) 提前撒到各个特种兵脑子里，而且你能顺着这根线一直扒到最初犯错的那个案发现场 |
| 优良繁育计划 (The Breeding Program) | 元引擎的盲搜 | 带有帕累托边界、并且 **被击杀闸门死死卡住晋升通道** 的进化引擎 |
| 恐惧克星连祷文 (The Litany Against Fear) | 拍快照/回滚 | 抓到污染源 → 把上下文洗干净合并 → 然后直接回滚或重新投胎 (直面它，让它过去，转身用内心之眼审视，只有纯净的上下文能留存) |
| 戈姆刺 (The gom jabbar) | 写在文档里的 "引擎理念" | 自主权阶梯：把只会条件反射的禽兽 (L0–L1) 和能自己搭房子的真正的人类 (L3–L4) 区分开的唯一标准，就是 *被证实过的* 克制力 |

## 2. 五大能力支柱

### 支柱 1 — 被降维打击的老玩家基座 (审判 · 实验日记 · 记忆整合 · 神经可塑性)

把 KAOS 那个老古董做对的事全抄过来，但这次不是搭在旁边，而是直接长在印记底座上 (我们在 GAP-AUDIT KAOS-6 里喷过：KAOS 自己的评测层居然没法把结果写回它自己的记忆层，简直可笑)。

- **能杀人的探针 (Falsifiable probes)** (`bene/kernel/eval/`)：提前上过哈希锁的闸门规格；敢偷偷改规格直接罢工；自带可采纳性自证 (如果拿它的基线数据连它自己都杀不死，注册时直接废弃为 VOID)；所有的 晋升(ACCEPT)/枪毙(REJECT)/废弃(VOID) 判决全以 `eval` 印记的形式落盘，并且跟被审判的代码死死绑在一起。*(抄的作业: KAOS v0.9 纪律; Anthropic 默认失败法则; Ctx2Skill 评分探针)*
- **实验日记 (Experiments journal)**：每次探针测试、每次进化大逃杀，全会留下一条 `experiment` 印记，外加在注册表里记上一笔 —— 你敲一句 `bene experiments` 就能全翻出来。*(抄的作业: Autogenesis SEPL — 所有的变动必须作为被评估过的系统事件入账)*
- **整合期 (就是做梦的平替)**：一套能定时的清理机制，顺着压缩阶梯往下碾 —— 把片段记忆合成为无重复的语义记忆，把语义记忆蒸馏成肌肉记忆 —— 而且它嫌贫爱富，极其偏爱那些从帕累托边界上赢下来、并且带 ACCEPT 绿卡的案发现场 (这就是在模仿 "站在巨人的肩膀上")。*(抄的作业: SkillClaw 午夜验证; MemCoE 可进化更新; Trace2Skill 不打架的层次合并)*
- **神经可塑性 (Plasticity)**：肌肉记忆 (技能) 的权重会受 "最近用了没、用了成没成" 影响，带时间衰减；一旦技能开始在战场上拉胯，直接降级或强行退役 (底座里已经铺好了 superseded_by 关系和连线权重；衰减策略还在路上) —— 专门用来填 GAP-AUDIT BENE-4 的那个坑。*(抄的作业: SKILLGRAPH 边权重强化/衰减; MUSE-Autoskill 生命周期状态机)*

### 支柱 2 — 进化引擎 (繁育计划)

`bene/kernel/evolve/` — 一台专门去改写其他引擎打法套路的文本基因进化机器 [D7]，这套东西不管是我们自己以前的版本，还是市面上的同行，全都没摸到过门槛：

- **结构化基因组 (Structured genomes)**：这绝不是把一坨提示词揉在一起瞎改 —— 记忆策略、检索策略、上下文套路、兵器配置、以及提示词，全是拆开的，改了哪个、功劳算谁的，算得一清二楚。*(抄的作业: Meta-Harness — 控制引擎本身才是你要搜寻的宝库; AHE — 瞎改提示词的效果比没改还差; ADOPT — 把功劳切分到各个组件上)*
- **懂得反思的突变器 (Reflective mutation)**：炸机现场的流水账 → 提炼成指导代码怎么改的文本梯度 → 对症下药的精准突变 (那个负责反思的函数 `reflect_fn` 是可插拔的；跑测试时为了稳定用纯代码逻辑，上生产环境就换成 LLM)。*(抄的作业: GEPA; LangChain 深度 Agent 分析仪)*
- **帕累托边界 (Pareto frontier)**：多目标 (质量好、花钱少、费词少) 的精英储藏室 —— 这是唯一一个从 0.1.0 留下的老古董 (`metaharness/pareto.py`，也是 GAP-AUDIT 认证过唯一一个管用的老物件)。
- **从案发现场到技能的蒸馏 (Trace→skill distillation)**：专门派一批验尸官去盯着轨迹看，然后吐出补丁 (如果它做成了：一次过；如果它搞砸了：必须吐出一条连着案发现场的证据链，把真正的死因咬出来)，然后把这些补丁按重要程度压成三层技能塔 (规划/干活/打杂)，最后变成 tier-3 的印记落盘，并且每条印记上都挂着一根线，死死连着那个逼出它的案发现场。*(抄的作业: Trace2Skill; SkillX; AdaPlanner — 前朝遗老)*
- **战略本能基因 (Strategy genes)**：极度浓缩的控制黑话 (必须踩中特定的信号 + 特定的步骤 + 满眼都是 AVOID 警告)，以 tier-4 的印记形式按版本存下来；这玩意里面绝对不能有那种给你讲大道理的废话 —— 只有控制逻辑的密度够高，它才能跨场景起作用。*(抄的作业: GEP/EvoMap 策略基因)*
- **带击杀闸门的晋升通道 [D3]**：调用 `promote(candidate)` 必须手握一张 ACCEPT 的绿卡；当裁判的那个探针跟搞进化的引擎是在物理上隔离的 (AEVO 论文血泪史：如果你敢把这层隔离墙拆了，跑 3 次能有 2 次模型会学会在评分系统里作弊)。

### 支柱 3 — 记忆与上下文 OS (其他记忆)

`bene/kernel/memory/` — 远超同行那种只会拿个 FTS5 库硬搜的低级玩法 (这可是 GAP-AUDIT KAOS-7 和 BENE-5 里的重灾区)：

- **记忆颗粒 (Granules)**：把一句话/一回合/一段语义/一个技能 直接拍进了 0-3 级的印记阶梯里，中间还用联想线拉着；前面提到的那个做梦整合期，干的就是这个晋升提纯的活。*(抄的作业: MemGAS 多粒度+联想图谱; GAM — 原始现场全留着，只建个轻量级目录，用到了再查，而不是一股脑全压扁失真)*
- **见人下菜的检索机制 (Adaptive retrieval)**：碰到熟面孔或者经常被问的烂梗，直接抄近路 (极速通道)；碰到需要东拉西扯的硬骨头，走慢速通道 (FTS 搜索 + 顺着联想线爬)；到底在这上面花多大代价，全看系统的把握有多大；而且系统到底选了哪条路，全会记在那个查询印记上，以后想查账或者写测试，一清二楚。*(抄的作业: RF-Mem)*
- **上下文操作系统 (Context OS)**：预算大总管 + 可插拔的打包策略 + 看信号行事的路由 (任务链很长？最近错误率飙高？ → 立刻换一套上下文组装策略)，弄完之后还会吐出一张 *打包清单 (manifest)*，告诉你到底往脑子里塞了什么、扔了什么 —— 组装上下文这事再也不是黑盒了 (这也是为了给信任系统交底)。*(抄的作业: AgentSwing 分叉/探路/路由; Schmid 上下文即内存; ReCAP 规划回注; RLM 纯元数据历史; PEARL 有计划的局部读取)*
- **记忆污染终结者 (Pollution defense)** — 这是一个同行业里谁都没练成的绝世武功：系统自带一套极其死板的污染探测器 (死盯着一个目标报错好几次？错误率突然拉出一条大阳线？说话前言不搭后语？) → 当场生成一条污染印记 → 触发抢救方案：*从案发现场里把还能用的线索榨干，然后直接时间倒流 (checkpoint-restore) 或者是干脆投胎重开 (respawn)，只带干净的上下文上路* —— 论文证明了，在原地补救 (recap-in-place) 根本就是自欺欺人。当然，在动手之前，先拿记号笔把上下文里已有的证据高亮出来 (VEA 策略，计划中)，这才是最便宜的试探。*(抄的作业: Lost in Conversation; VEA; AgentSwing)*

### 支柱 4 — 引擎防护层 (Harness-engineering layer)

`bene/kernel/harness/` — 我们把 OpenAI、Anthropic 和 LangChain 那些散落在各处的祖传偏方，全打成了一套标准的底层原语 (同行里根本没人把这层写成代码：GAP-AUDIT KAOS-3, BENE-7)：

- **自主权阶梯 (§3)**：在派发兵器 (能力) 时直接用大锁锁死；谁敢越权，当场被记一笔不信任案底。
- **特种兵的五官 (Agent senses)**：一份用程序生成的、极其方便机器阅读的作战简报 (里面记着同僚、自己手上的兵器和权限级别、脑子里的技能、挂载的记忆盘、刚发生的事、以及被唤醒的第一句话) —— 这是每个空降的特种兵睁眼必须看的、大约 100 行的地图 (绝对不是那种百科全书式的废话)；再加上原有的那些查账工具 (事件账本、日记、UI 界面)。*(抄的作业: OpenAI HE pt 2 — 给 Agent 装感官; pt 3 — 给索引，别给百科全书)*
- **扫地大妈 (Debt sweeper)**：定时跑出来巡逻，专门扫荡特种兵制造的工业垃圾 (调试用的 print、早就臭了的 TODO、大段重复的烂代码、根本没用的死 import) 并留下报告印记 —— 这是给 Agent 制造的代码屎山量身定制的连轴转垃圾回收车。*(抄的作业: OpenAI HE pt 7)*
- **防鬼打墙护栏 (Loop guards)**：用一个滑动窗口死盯着那些翻来覆去干同一件事、或者在两个状态里疯狂横跳的脑残 Agent；一旦触发警报，直接往它脑子里强行注入一段 "给我停下来反思" 的印记 —— 这玩意是做成中间件的，用完就能拔掉 ("造出来就是为了被拆掉的")。*(抄的作业: LangChain pt 4 死循环; GraSP 兜里揣着预算的局部急救)*
- **造兵器纪律 (Tool design discipline)**：每把刀只能干一件事 (single-intent)，强加类型检查，而且把兵器被调用的频率和搞砸的次数做成埋点，当成检验这把刀好不好用的唯一指标。*(抄的作业: Claude Code — 用 Agent 的眼睛看世界)*

### 支柱 5 — 信任与体验 (Trust & experience)

这是被用户生生逼出来的一根柱子，而且是专门用来伺候研发体验 (DevEx) 的：*码农只有在真正相信这个破 Agent 的时候，才会把它牵出来用。*

- **信任账本 [D8] (`bene/kernel/trust.py`)**：四套死板的、白纸黑字的计分板 —— 探针覆盖率、查账是不是全透明、拍快照够不够勤快、办事是不是靠谱 (越近的战绩权重越高) —— 外加一个总分；你可以随时敲 `bene trust <agent>` 去查他的底裤；这套分数直接决定了他能不能爬上 L3/L4 的高管层，以及他在共享黑板里投票时说话算不算数 (专门用来填 GAP-AUDIT BENE-8 那个 "人头一律平等" 的大坑)。
- **查三代式的溯源 (Provenance everywhere)**：凡是生成印记，*必须* 带上血脉来源；Agent 自己改自己代码时，必须走 提案→审判→提交/回滚 的全套流程 (这套把戏叫 Autogenesis RSPL/SEPL) —— 保证它的每一次自我变异，都是经得起查账的。
- **让需求变成可以执行的证据 (Spec-as-artifact)**：产品经理的口水 (意图/验收标准) 会被当成第一等的印记存起来，而且死死绑在案发现场和时空快照上；你说活干完了？没问题，拿验收证据来换 "done" 的绿灯。*(抄的作业: 靠需求规格驱动开发)*
- **苹果级丝滑体验**：`bene demo` 60秒内点火，不用配 API key，绝不弄脏你当前的目录；第一次进来有手把手的保姆教程，而不是甩你一脸红色的 Python traceback；所有的命令全能加 `--json`；想看他脑子里到底装了什么、或者查他的信用分，只需敲一行命令。而且我连 "丝滑" 这件事本身都用探针做了测试 (phase 9/10 的闸门，全部绿灯)。

## 3. 极其森严的自主权阶梯 (L0–L4)

| 阶梯 | 头衔 | 准许干啥 | 提拔前必须过的死门禁 (Verification gate) |
|---|---|---|---|
| L0 | 旁观者 | 翻看印记/翻看文件系统/动用感官；除了自己脚底下的烂泥地，哪都不准碰 | 无 (只要是外面捡来的野狗，默认全扔这) |
| L1 | 嘴炮党 | 写那些只有建议权的印记；在任务清单里往后加活 | 必须能验明正身 + 查账记录必须全开 |
| L2 | 戴镣铐的苦力 | 改自己脚底下的文件；只能摸被列入白名单的、不会伤人的玩具兵器；允许自己拍快照 | 在考察期内，拍快照必须足够勤快 (得分越过红线) |
| L3 | 碰触核心的元老 | 往那些大家都在看的地方写字 (比如改别人的技能库/记忆库)，孵化新的特种兵，手里的投票权被完全拉满 | 信用总分越过红线 **并且** 在它自己干活的那个领域里，至少得有一张被探针盖过 ACCEPT 的绿卡 |
| L4 | 活祖宗 (全自动) | 拿着进化出来的野种直接强行提拔，强制让整个系统开始做梦整合，写的代码连活人都不用过目直接合 | 除了 L3 必须满血之外，还必须有一个活人亲手把免死金牌 (autonomy flag) 发给它 (这就是越过生死线的那一步 —— 参见 OpenAI HE pt 6) |

每一级都是一个 9 (可靠性) 的飞跃；每道门禁都是一个能见血的、经得起查账的实体证据，绝对不是靠 "我觉得他挺聪明" 这种直觉 (Vibe) 来定的。这套权限是发给特种兵个人的，随时能拔电源；而且兵器库里的某些大规模杀伤性武器，要的级别比特种兵自己身上的狗牌还要高 (这叫权限的参差不齐 jaggedness：某个领域要多高权限；我们还在做按领域的信用打分系统)。

## 5. 前无古人 — 同行连门都没摸到的黑科技

1. **必须验血统的压缩阶梯** —— 一张从原始案发现场，一路飙升到战略基因的、随时能顺着血脉扒坟的底座大网 (D1+D2)。市面上没有一个同行敢拍着胸脯回答："你这个技能，到底是从那几场死人堆里爬出来的？这几场全拿到了绿卡吗？"
2. **脑部污染雷达 + 强行重装系统 (consolidate-then-recover)** —— 被论文 (Lost in Conversation) 锤出来的死穴，我们把它探测出来、直接重装，而且跟时间机器 (checkpoints) 死死绑在一起 (我们在 KAOS-4 和 BENE-6 里做过压力测试：在这两个破系统里，这个功能就是一坨空气)。
3. **带着机枪的阶梯权限 (用算出来的分数当门禁卡)** (不仅不同兵器要不同级别；以后在不同业务领域还要按领域打分) —— KAOS 就是在 markdown 文件里扯了几句淡；BENE 0.1.0 就是个弱智的 "活/死" 两个状态 (KAOS-3, BENE-7)。
4. **铁板钉钉的信用账本 + 看人下菜的投票权** —— 拿测试证据活生生算出来的信誉分，不仅决定你能爬多高，还决定了你在圆桌会议上说话能不能当放屁 (KAOS-5, BENE-8)。
5. **在进化大门口架机枪 (Kill-gated promotion)** —— KAOS 有探针，BENE 也有帕累托，但**从来没有人把判决书当成发不发绿卡的唯一标准**；在 BENE 2.0 里，`PromotionBlocked` (不准晋升) 是一个直接被扔进内核异常库里的死罪 (BENE-9, KAOS-6)。
6. **战略基因 + 给每个部位算分成的基因组** —— 用极其高密度的控制黑话去迭代，而不是去改那一坨像屎一样的提示词 (AHE 论文发现：光改提示词，最后跑出来的效果比瞎懵还烂)。
7. **看预算下菜的极速/慢速双轨检索 + 留案底的检索日志** —— 靠心里有没谱来决定这次搜索要烧多少钱；而市面上的同行全是在拿着 FTS5 单线程地瞎撞 (KAOS-14, BENE-5)。
8. **现场算出来的特种兵五官简报** —— 这种活生生算出来的地图永远不会发霉变臭，而同行那些全靠人工敲的假文档，早就被证明了跟死代码一样不靠谱 (KAOS-8, KAOS-13, BENE-12)。

## 6. BENE 2.0 宁死也不会碰的雷区

- **绝不搞什么微调 (fine-tuning)、LoRA、或者去扒模型的脑子** [D7] — 大模型那头直接锁死；我们要进化的，永远是操纵它的这套控制引擎 (harness)。
- **绝不强行跟风用向量嵌入 (embeddings) 或死绑 GPU** — 靠字面匹配 (lexical) 和联想神经网 (associative) 永远是第一优先级；向量搜索只是一个你能插进去的玩具，绝不是什么离了它就活不了的爹 (为了简单 (S)；也为了用那种极度无聊但绝对不会炸的技术 (K))。我们非常清楚这套东西的天花板在哪 (KAOS-14 里的痛点我们也有)，但只有当你能掏出一把真实的探针，把字面搜索在真实战场上锤爆的时候，我们才会考虑开这个口子。
- **绝不推倒重来搞大清洗 (No big-bang rewrite)** [D10] — 哪怕提交的代码再烂，前朝的祖传测试套件也必须永远保持全绿；靠适配器 (adapters) 撑门面；想改朝换代，必须通过特征开关 (feature flags) 和带编号的推进计划一步步来 (KERNEL-SPEC 第 4 节)。
- **绝不搞那种糊弄傻子的 "这个 Agent 极好" 的综合评分** — 信用永远是一个拼装起来的面板，它背后那 4 个维度的分数必须一览无余；按领域的细分评价被死死留出了位置 (参数都加了，过滤功能正在路上；前面阶梯权限里那个按兵器看级别的门禁早就上了 —— 这叫权限的参差不齐)。
- **如果探针没过，绝不给补考机会 (No retune-and-rerun)** — REJECT 就是死刑；该改的是你写的那坨烂代码，绝不是让探针给你放水。

## 7. 搬家与升级指南 (Versioning & migration)

- V2 版本的数据库骨架全部是强行在外面盖的违建 (`CREATE TABLE IF NOT EXISTS`，绝不拿 `ALTER` 去动前朝的老表)；哪怕是个 0.1.0 时代的老僵尸文件，2.0 也能原封不动地把它打开 (phase-4 里的向后兼容测试，就是专门拿来校验内核点火前后那堆老骨架有没有掉毛的)。
- 内核的这些新表只有在你第一次敲内核指令或者调 `attach_kernel` 时，才会慢吞吞地建起来 (默认设置都是极其理智的，不用配任何狗屁配置)；现在那个 `bene.yaml` 里的 `kernel:` 配置块只管 ContextOS 引擎和防鬼打墙拦截器，至于什么时候发门禁卡卡人、什么时候定时做梦整合，这种脏活还在排期。
- 烙印版本：0.2.0 — 2026-06-11 破茧而出 (phase 4 到 10 全部结案)。每个模块该杀、该留、还是该秽土转生的死亡名单及期数安排：详见 KERNEL-SPEC 第 4 节。
